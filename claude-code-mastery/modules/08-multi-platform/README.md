# Module 08: Multi-Platform Claude Code

**Module 08, Overview — Claude Code Mastery**

**Instructor: Jabbir Basha, Principal AI Engineer**

---

## Overview

Claude Code is not limited to the terminal. It is available across multiple platforms — in the browser, inside VS Code, within JetBrains IDEs, and as a standalone desktop application. This module walks you through each platform, explaining how to set up, configure, and get the most out of Claude Code regardless of where you prefer to work.

## Prerequisites

- Completed Modules 01–07 (core CLI concepts, memory system, hooks)
- An Anthropic account with an active Claude plan
- At least one of the target platforms installed (VS Code, JetBrains IDE, or a modern browser)

---

## Learning Objectives

By the end of this module, you will be able to:

1. **Use Claude Code in the browser** via claude.ai/code and understand its sandbox environment
2. **Install and configure the VS Code extension** for seamless editor-integrated AI assistance
3. **Set up Claude Code in JetBrains IDEs** including IntelliJ, PyCharm, and WebStorm
4. **Work with the Desktop App** on Mac and Windows for a native experience
5. **Choose the right platform** for a given task based on trade-offs and capabilities

---

## Platform Comparison

| Feature | CLI | Web (claude.ai/code) | VS Code Extension | JetBrains Extension | Desktop App |
|---------|-----|----------------------|-------------------|---------------------|-------------|
| Local file access | Yes | No (cloud sandbox) | Yes | Yes | Yes |
| Requires install | Yes | No | Yes | Yes | Yes |
| Terminal integration | Native | Sandboxed | VS Code terminal | IDE terminal | Built-in |
| MCP server support | Yes | Limited | Yes | Yes | Yes |
| Hooks support | Full | SessionStart only | Full | Full | Full |
| Offline capable | No | No | No | No | No |
| Best for | Power users | Quick tasks, no setup | VS Code developers | JetBrains developers | General use |

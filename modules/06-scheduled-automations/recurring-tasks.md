# Creating Recurring Tasks

In this lesson, you will learn how to set up recurring instructions in Claude Cowork — tasks that run automatically on a schedule without you having to ask each time.

---

## What Are Recurring Instructions in Cowork?

A recurring instruction is a task that you define once and schedule to repeat at regular intervals. Instead of manually telling Cowork to do the same thing every day or every week, you set up the instruction with a frequency, and Cowork takes care of the rest.

Think of it like setting an alarm clock, but instead of waking you up, it tells Cowork to get to work on a specific task.

---

## Setting Up Your First Recurring Task

Creating a recurring task is straightforward:

1. **Write your instruction** — Describe the task clearly, just as you would for a one-time request
2. **Choose a frequency** — Tell Cowork how often to run the task
3. **Set the start time** — Specify when the first run should happen
4. **Confirm and activate** — Review the setup and turn it on

**Example instruction:**

> "Every Monday morning, check my Downloads folder and organize any new files into the appropriate subfolders based on file type (documents, images, spreadsheets, and other)."

This single instruction will run every Monday, keeping your Downloads folder tidy without any effort on your part.

---

## Frequency Options

Cowork supports several scheduling frequencies to match your needs:

| Frequency | Description | Example Use Case |
|-----------|-------------|------------------|
| **Daily** | Runs once every day at your chosen time | Checking for new emails or files |
| **Weekly** | Runs once a week on a specific day | Compiling a weekly summary report |
| **Monthly** | Runs once a month on a specific date | Generating end-of-month financial summaries |
| **Custom** | Runs on a schedule you define (e.g., every 2 days, every other Tuesday) | Bi-weekly inventory checks |

Choose the frequency that best fits how often the task needs to happen. When in doubt, start with a weekly schedule and adjust from there.

---

## Example: Weekly File Organization

> "Every Monday morning, check my Downloads folder and organize any new files into the appropriate subfolders based on file type (documents, images, spreadsheets, and other)."

**What Cowork does each Monday:**
- Opens the Downloads folder
- Identifies any new files added since the last run
- Moves each file to the correct subfolder
- Logs what was moved and where

**Why this works well as a recurring task:**
- It is repetitive and predictable
- It follows clear rules
- It does not require your input each time

---

## Example: Weekly Invoice Summary

> "Every Friday, compile all invoices from the week into a summary report. Include the vendor name, invoice amount, and date received. Save the report to my Weekly Reports folder."

**What Cowork does each Friday:**
- Gathers all invoices received during the week
- Extracts the key details from each invoice
- Creates a clean summary report
- Saves it in the designated folder

This type of recurring task saves significant time for anyone who processes invoices regularly.

## Managing Recurring Task Schedules

Once you have recurring tasks set up, you can manage them easily:

- **View all active tasks** — See a list of everything that is currently scheduled
- **Edit a task** — Update the instruction, change the frequency, or adjust the timing
- **Delete a task** — Remove a recurring task you no longer need

It is a good practice to review your active tasks periodically to make sure they are still relevant and working as expected.

---

## Pausing and Resuming Recurring Tasks

Sometimes you need to temporarily stop a recurring task without deleting it entirely:

- **Pause** — Stops the task from running on its next scheduled time. The task and all its settings are preserved.
- **Resume** — Reactivates a paused task so it starts running again on schedule.

**When to pause instead of delete:**
- You are going on vacation and do not need the task for a few weeks
- You want to troubleshoot an issue before the next run
- The task is seasonal and only needed during certain months

---

## Best Practices for Reliable Recurring Automations

Follow these guidelines to get the most out of your recurring tasks:

1. **Be specific in your instructions** — The clearer your instruction, the more consistent the results will be across repeated runs
2. **Start simple** — Begin with a straightforward task before building complex automations
3. **Test before scheduling** — Run the task once manually to make sure it works as expected before setting it to recur
4. **Use consistent folder structures** — Keep input and output files in predictable locations so Cowork always knows where to look
5. **Review results regularly** — Check the output of your recurring tasks periodically, especially in the first few weeks
6. **Set realistic frequencies** — Do not schedule tasks more often than necessary. A weekly check is often sufficient where a daily check might be overkill.

---

## Summary

Recurring instructions let you automate repetitive tasks by defining them once and scheduling them to run on a regular basis. Whether it is organizing files, compiling reports, or performing routine checks, Cowork handles it automatically so you can focus on more important work.
